# ms02

Non-Spark dev host. Runs Docker CE (buildx, compose), Kind, `act`, and remote IDE.

| Field | Value |
|---|---|
| LAN IP (wired `eno4`) | `192.168.1.189` |
| Ansible group | `dev_hosts` |
| Ansible user | `root` |
| Login user for kind / Tilt | `casibbald` |
| Playbooks | `playbooks/dev_hosts.yml`, `playbooks/docker_dev_engine.yml`, `playbooks/ms02_reverse_tunnel.yml` |
| Mac SSH aliases | `ms02` (casibbald, LAN), `ms02-root`, **`ms02-away`** (via Hostinger reverse tunnel) |

## Reachability from the Mac

**LAN-first:** **`192.168.1.189`** (Hermes **9119**, Tilt **10348**, NFS **2049**, kind API **38839**, …).
`just ms02-lan-check` probes common HTTP ports.

**Away from LAN (2026-05-31+):** outbound reverse SSH tunnel to
[srv1719193](./srv1719193.md) — `ssh ms02-away` or
`ssh -p 22002 casibbald@76.13.1.95`. Service: `ms02-reverse-tunnel.service`
(autossh, systemd `Restart=always`). See
[docs/ms02-reverse-tunnel.md](../../docs/ms02-reverse-tunnel.md).

**Retired:** `dev-tunnel-*` / `ms02-dev-tunnel` (Starlink Wi-Fi↔LAN workaround) —
see [starlink-wifi-lan-port-filter](../concepts/starlink-wifi-lan-port-filter.md).

## Relevance to Sparks

- Build artifacts / Kind clusters for CI-like work land here, not on the Sparks.
- When we need a "client" to hit the vLLM API (`curl`, OpenAI Python SDK, eval harness),
  ms02 or the operator's laptop are the right places — keeping the Sparks doing only
  serving.

See [`docs/dev_hosts.md`](../../docs/dev_hosts.md) for the operator guide and
[`docs/docker-dev-host.md`](../../docs/docker-dev-host.md) for the original
layout notes.
