# ms02

Non-Spark dev host. Runs Docker CE (buildx, compose), Kind, `act`, and remote IDE.

| Field | Value |
|---|---|
| LAN IP (wired `eno4`) | `192.168.1.189` |
| Ansible group | `dev_hosts` |
| Ansible user | `root` |
| Login user for kind / Tilt | `casibbald` |
| Playbooks | `playbooks/dev_hosts.yml`, `playbooks/docker_dev_engine.yml` |
| Mac SSH aliases | `ms02` (casibbald), `ms02-root` (root) — operator-maintained under `~/.ssh/config.d/`; **`ms02-dev-tunnel` retired** (removed by `mac-provision` if present) |

## Kind cluster (running)

| Field | Value |
|---|---|
| Cluster | `kind` (context `kind-kind`) |
| Control plane container | `172.19.0.2` (docker network `br-…`) |
| kube-apiserver (host-mapped on ms02) | `https://192.168.1.189:38839` from the Mac (or `https://ms02:38839`) — use LAN URL in kubeconfig |
| Tilt UI | `0.0.0.0:10348` (v6-only socket — localhost fallback works) |

## Reachability from the Mac

**LAN-first:** **`192.168.1.189`** (Hermes **9119**, Tilt **10348**, NFS **2049**, kind API **38839**, …).
`just ms02-lan-check` probes common HTTP ports. **`dev-tunnel-*`** / **`ms02-dev-tunnel`**
are **retired** — see [starlink-wifi-lan-port-filter](../concepts/starlink-wifi-lan-port-filter.md) for history.

## Relevance to Sparks

- Build artifacts / Kind clusters for CI-like work land here, not on the Sparks.
- When we need a "client" to hit the vLLM API (`curl`, OpenAI Python SDK, eval harness),
  ms02 or the operator's laptop are the right places — keeping the Sparks doing only
  serving.

See [`docs/dev_hosts.md`](../../docs/dev_hosts.md) for the operator guide and
[`docs/docker-dev-host.md`](../../docs/docker-dev-host.md) for the original
layout notes.
